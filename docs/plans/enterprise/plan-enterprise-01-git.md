# 开发计划：Git 版本管理（plan-enterprise-01-git）

## 1. 概述

本模块为工作流提供 Git 级别的版本管理能力，解决大型企业对工作流变更可追溯、可对比、可回滚的诉求。覆盖工作流导出到 Git 仓库、版本对比（diff）、回滚到任意历史版本、分支管理，以及工作流版本历史（自动保存每次变更、版本对比、快速回滚）。

不覆盖范围：

- 协作编辑的实时冲突解决（见 [plan-enterprise-02-collab.md](plan-enterprise-02-collab.md)）。
- 凭据、密钥等敏感信息的版本管理（凭据不进入 Git 明文存储）。
- Git 仓库本身的运维（由企业内部 Git 平台承担）。

## 2. 交付物清单

- 工作流导出到 Git 仓库（JSON 序列化 + 提交）。
- 版本对比（diff）接口与前端可视化。
- 回滚到任意历史版本。
- 分支管理（创建、切换、合并、删除）。
- LibGit2Sharp 或 Git 命令集成（可配置切换）。
- 工作流版本历史（自动保存每次变更、版本对比、快速回滚）。
- 单元测试与集成测试。

## 3. 开发阶段

### 阶段一：Git 集成基础

- 目标：打通工作流到 Git 仓库的导出链路，支持提交与拉取。
- 核心任务：
  - 引入 LibGit2Sharp 或 Git 命令集成（通过配置切换）。
  - 实现工作流 JSON 序列化与导出到 Git 仓库。
  - 实现提交（commit）与基础仓库初始化。
  - 配置 Git 仓库连接（远程地址、认证凭据引用）。
- 输入：GA 阶段工作流 CRUD、凭据系统。
- 输出：可导出工作流到 Git 仓库并生成提交记录。
- 验收标准：
  - 工作流可导出为 JSON 并提交到配置的 Git 仓库。
  - 提交记录包含操作人、时间、变更摘要。
  - Git 仓库认证凭据通过凭据 ID 引用，不硬编码。
- 依赖：GA 阶段工作流 CRUD、MVP 凭据系统。

### 阶段二：版本对比与回滚

- 目标：支持任意两个版本之间的 diff 对比，以及回滚到指定历史版本。
- 核心任务：
  - 实现版本对比（diff）接口，输出节点级与参数级差异。
  - 前端可视化 diff（新增/删除/修改节点高亮）。
  - 实现回滚到任意历史版本（生成新提交，不破坏历史）。
  - 回滚前校验工作流是否被触发器引用，给出告警。
- 输入：阶段一 Git 集成基础。
- 输出：版本对比与回滚能力。
- 验收标准：
  - 任意两个版本可查看节点级与参数级差异。
  - 可回滚到任意历史版本，回滚后工作流可正常加载执行。
  - 回滚生成新提交，历史记录完整保留。
- 依赖：阶段一。

### 阶段三：分支管理

- 目标：支持分支创建、切换、合并、删除，支撑多团队并行开发。
- 核心任务：
  - 实现分支创建、切换、列出、删除。
  - 实现分支合并（merge）与冲突检测。
  - 分支权限控制（依赖 GA 阶段 RBAC）。
  - 前端分支管理界面。
- 输入：阶段二、GA 阶段 RBAC。
- 输出：分支管理能力。
- 验收标准：
  - 可创建分支并在分支上独立保存工作流变更。
  - 分支合并时检测冲突并提示处理方式。
  - 分支操作受 RBAC 权限控制。
- 依赖：阶段二、GA 阶段 RBAC。

### 阶段四：工作流版本历史

- 目标：在工作流编辑器内提供自动保存与版本历史能力，无需手动提交 Git。
- 核心任务：
  - 编辑器自动保存每次变更（可配置间隔与最大版本数）。
  - 版本历史列表与时间线展示。
  - 版本对比（复用阶段二 diff 能力）。
  - 快速回滚（复用阶段二回滚能力）。
  - 自动保存版本与 Git 提交版本的关联（自动保存可升级为 Git 提交）。
- 输入：阶段二。
- 输出：工作流版本历史能力。
- 验收标准：
  - 编辑器自动保存每次变更，版本历史可追溯。
  - 可从版本历史快速回滚到任意版本。
  - 自动保存版本可手动升级为 Git 提交。
- 依赖：阶段二。

## 5. 风险与待定项

| 风险/待定项 | 影响 | 应对策略 |
|------------|------|---------|
| LibGit2Sharp 在 Windows 环境原生库兼容性 | 集成失败 | 提供 Git 命令回退方案，通过配置切换 |
| 大型工作流 diff 性能 | 对比延迟 | 节点级 diff 优先，参数级 diff 按需加载 |
| 分支合并冲突处理复杂 | 用户体验差 | 先提供冲突检测与提示，复杂冲突引导手动解决 |
| 自动保存版本数量膨胀 | 存储压力 | 配置最大版本数，超限自动归档旧版本 |
| Git 仓库认证凭据轮换 | 集成中断 | 凭据通过 ID 引用，轮换后自动生效 |

## 6. 验收总标准

- 工作流变更可回滚到任意历史版本（roadmap §6 核心验收项）。
- 版本对比可查看节点级与参数级差异。
- 分支管理支持多团队并行开发，受 RBAC 控制。
- 自动保存与 Git 提交双向关联。
- 单元测试覆盖率 ≥ 80%。
