# 开发计划：执行清理（plan-beta-06-execution-cleanup）

## 1. 概述

为 Flow Engine 引入执行记录定时清理能力，避免执行历史无限堆积导致数据库膨胀。本模块提供后台 HostedService 按配置化保留策略自动清理过期执行记录。

### 1.1 覆盖范围

- 定时清理过期执行记录（后台 HostedService）。
- 配置化保留策略（按时间、按数量）。
- 清理审计记录。

### 1.2 不覆盖范围

- 执行记录归档到外部存储（不在 Beta 范围）。
- 清理工作流定义（仅清理执行记录）。
- 清理审计日志文件（由审计日志模块管理）。

## 2. 交付物清单

- 清理后台服务（IHostedService 实现）。
- 保留策略配置（按时间保留天数、按数量保留条数）。
- 清理执行逻辑（按策略删除过期 ExecutionRecord）。
- 清理审计事件（记录清理操作）。
- 清理调度配置（间隔、启用开关）。
- 单元测试与集成测试。

## 3. 开发阶段

### 阶段一：清理后台服务

- 目标：实现定时清理过期执行记录的后台服务。
- 核心任务：
  - 实现 ExecutionCleanupHostedService（IHostedService）。
  - 按配置间隔定时扫描过期执行记录。
  - 删除超过保留期限的 ExecutionRecord 及关联 NodeExecutionRecord。
  - 清理过程不影响在线执行（仅清理已完成/失败/取消的记录）。
  - 清理操作记录审计事件。
- 输入：MVP 持久化层（ExecutionRecord 存储）。
- 输出：清理 HostedService、清理逻辑。
- 验收标准：
  - 后台服务按间隔定时执行清理。
  - 过期执行记录被删除。
  - 在线执行不受影响。
  - 清理操作有审计记录。
- 依赖：MVP 持久化。

### 阶段二：保留策略配置

- 目标：保留策略可配置，支持按时间与按数量。
- 核心任务：
  - 定义保留策略配置（RetentionByDays、RetentionByCount）。
  - 按时间：删除创建时间超过 N 天的记录。
  - 按数量：保留最近 N 条记录，删除多余。
  - 两种策略可组合（先按时间，再按数量）。
  - 配置热加载支持（可选）。
  - 清理调度间隔与启用开关配置。
- 输入：阶段一清理服务。
- 输出：保留策略配置模型、策略执行逻辑。
- 验收标准：
  - 按时间策略：超过 N 天的记录被清理。
  - 按数量策略：仅保留最近 N 条。
  - 策略可组合生效。
  - 配置可调整。
- 依赖：阶段一。

## 5. 风险与待定项

| 风险 | 影响 | 应对 |
|------|------|------|
| 清理误删在线执行 | 执行中断 | 仅清理已完成/失败/取消状态记录，状态校验 |
| 大批量删除锁表 | 数据库阻塞 | 分批删除，单次限量 |
| 待定：归档到外部存储 | 影响合规留存 | Beta 直接删除，归档延后 |
| 待定：按项目差异化保留 | 影响多租户公平性 | Beta 全局策略，按项目策略延后 |

## 6. 验收总标准

- 过期执行记录自动清理。
- 保留策略可配置（按时间、按数量、组合）。
- 清理不影响在线执行。
- 清理操作有审计记录。
- 单元测试覆盖率 ≥ 70%，集成测试覆盖清理场景。
